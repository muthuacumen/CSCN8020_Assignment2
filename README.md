## Summary

One Jupyter Notebook implementing **Q-Learning** on the **Taxi-v3** Gymnasium environment
(500 states, 6 actions). The agent learns to pick up a passenger and deliver them to their
destination by updating a tabular Q-table after every step using the Temporal Difference
update rule.

The notebook covers:
1. **Q-Learning implementation** — `TaxiEnvironmentManager`, `QLearningAgent`,
   `QLearningTrainer`, `ExperimentRunner`, `MetricsLogger`, `PlotManager`
2. **Training metrics and plots** — reward/episode and steps/episode curves (smoothed)
3. **Hyperparameter experiments** — α ∈ {0.1, 0.2, 0.01, 0.001} and ε ∈ {0.1, 0.2, 0.3}
4. **Best combination** — α=0.2, ε=0.1 selected from experimental evidence
5. **Five talking points** — each mapping code to Sutton & Barto (2018) pseudocode
6. **Greedy policy evaluation** — post-training assessment with ε=0

## Q-Learning Algorithm

Q-Learning (Watkins, 1989; Sutton & Barto 2018, §6.5) is a **model-free, off-policy
temporal difference** control algorithm. After every step the agent applies:

```
Q(S,A) ← Q(S,A) + α [ R + γ · max_a Q(S′,a) − Q(S,A) ]
```

| Symbol | Meaning |
|--------|---------|
| α | Learning rate — how much new experience overwrites old Q-values |
| γ | Discount factor — how much to value future rewards (0.9 here) |
| R | Immediate reward received |
| max_a Q(S′,a) | Best Q-value in the next state (bootstrap estimate) |
| TD error | `[target − Q(S,A)]` — the "surprise" that drives learning |

## Design Notes

- **OOP**: six classes with clear separation of responsibilities — environment management,
  agent (Q-table + action selection + update), training loop, experiment orchestration,
  logging, and plotting.
- **Gymnasium API**: follows the `reset()` / `step()` 5-tuple pattern used in course
  materials (HelloGymMaze, playground lec2–lec4).
- **Reproducibility**: all randomness through seeded `numpy.random.default_rng(42)`;
  `env.reset(seed=42)` on first episode only so subsequent episodes vary naturally.
- **Logging**: Python `logging` module → `logs/sample_execution.log`, timestamped,
  records every 500-episode training interval and all experiment transitions.
